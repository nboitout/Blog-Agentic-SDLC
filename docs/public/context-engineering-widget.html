Context Engineering — The Conveyor Belt

Every turn replays the full conversation from scratch. Input tokens grow linearly per turn, so cumulative cost grows quadratically with conversation length. This widget makes the curve visible — and shows what deliberate context management changes.

Why quadratic, not exponential? Each response is roughly constant length (~400 tokens). The input grows by one exchange per turn. Cost per turn = (system + n × per_turn) × price. Summed over N turns: N·S + P·N(N+1)/2 — a polynomial of degree 2, not exponential.
Engineered mode caps context at the last 3 turns instead of the full history, simulating proactive context compression — summarisation, state extraction, or conversation splitting. The green curve is flat regardless of session length.
Try it: Set tokens/turn to 8,000 (realistic for agentic sessions with tool calls) and watch the window fill by turn 4. Then enable Engineered mode to see the difference.
